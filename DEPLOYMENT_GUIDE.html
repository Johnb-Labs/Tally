3. Manual Installation

Step 3.1: Server Initial Setup

Step 1.1: Connect to Your Server

# Connect to your Ubuntu 24.04 server ssh your-username@your-server-ip # Update system packages sudo apt update && sudo apt upgrade -y # Install essential tools sudo apt install -y curl wget git unzip software-properties-common ufw
Note: If you used the automated installation script, skip this manual section and go to Post-Installation Setup.

Step 3.2: Install Node.js 20

# Install Node.js 20 LTS curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs # Verify installation node --version && npm --version

Step 3.3: Create Application User

# Create system user for the application sudo useradd --system --home-dir /opt/tally --shell /bin/bash tally sudo mkdir -p /opt/tally sudo chown tally:tally /opt/tally

Step 3.4: Install and Configure PostgreSQL

# Install PostgreSQL 16 sudo apt install -y postgresql postgresql-contrib sudo systemctl start postgresql sudo systemctl enable postgresql # Create database and user sudo -u postgres psql -c "CREATE USER tally WITH PASSWORD 'your_secure_password';" sudo -u postgres psql -c "CREATE DATABASE tally OWNER tally;" sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE tally TO tally;"

Step 3.5: Install and Configure Nginx

# Install Nginx sudo apt install -y nginx sudo systemctl start nginx sudo systemctl enable nginx

Step 3.6: Install PM2

# Install PM2 globally sudo npm install -g pm2 # Setup PM2 to start on boot sudo pm2 startup systemd -u tally --hp /opt/tally

Step 3.7: Configure Firewall

# Enable and configure UFW firewall sudo ufw --force enable sudo ufw allow ssh sudo ufw allow 'Nginx Full' sudo ufw status verbose

Step 3.4: Configure PostgreSQL Authentication

Edit the PostgreSQL configuration to allow password authentication:

sudo nano /etc/postgresql/*/main/pg_hba.conf

Find the line that looks like:

local all all peer

Change it to:

local all all md5

Then restart PostgreSQL:

sudo systemctl restart postgresql

Step 3.5: Test Database Connection

psql -h localhost -U tally_user -d tally_db

You should be prompted for the password and then see the PostgreSQL prompt. Type \q to exit.

4. Post-Installation Setup

Step 4.1: Deploy Your Application

After installation (automated or manual), deploy your Tally application:

# Switch to application directory cd /opt/tally # Clone your repository (replace with your actual repository URL) sudo -u tally git clone https://github.com/yourusername/tally.git . # Create environment file from template sudo -u tally cp .env.example .env

Step 4.2: Configure Environment Variables

# Edit environment configuration sudo -u tally nano /opt/tally/.env

Update the following key variables:

# Database Configuration (use password from /tmp/database_config.txt) DATABASE_URL=postgresql://tally:your_db_password@localhost:5432/tally PGHOST=localhost PGPORT=5432 PGDATABASE=tally PGUSER=tally PGPASSWORD=your_db_password # Session Configuration (generate with: openssl rand -base64 64) SESSION_SECRET=your-super-secret-session-key-here # Replit Authentication Configuration REPL_ID=your-repl-id REPLIT_DOMAINS=your-domain.com ISSUER_URL=https://replit.com/oidc # Application Configuration NODE_ENV=production PORT=5000
Critical: If you used the automated script, check /tmp/database_config.txt for the generated database password.

Step 4.3: Run Deployment Script

# Run the deployment script (created by automated installer) cd /opt/tally ./deploy.sh

If the deployment script doesn't exist (manual installation), run these commands:

# Install dependencies and build sudo -u tally npm ci --production sudo -u tally npm run build # Initialize database sudo -u tally npm run db:push # Start application with PM2 sudo -u tally pm2 start ecosystem.config.js sudo -u tally pm2 save

Step 4.4: Verify Installation

# Check application status sudo -u tally pm2 status # Check application logs sudo -u tally pm2 logs tally # Check if application is responding curl http://localhost:5000

You should see the application running on port 5000.

Step 4.5: Configure Domain Name

If you have a domain name, update the Nginx configuration:

# Edit Nginx configuration sudo nano /etc/nginx/sites-available/tally # Replace 'server_name _;' with your domain # server_name your-domain.com; # Reload Nginx sudo systemctl reload nginx

5. SSL Certificate Setup

Step 5.1: Install Certbot

# Install Certbot for Let's Encrypt SSL certificates sudo apt install -y certbot python3-certbot-nginx

Step 5.2: Get SSL Certificate

# Get SSL certificate (replace your-domain.com with your actual domain) sudo certbot --nginx -d your-domain.com # Or for both www and non-www versions: sudo certbot --nginx -d your-domain.com -d www.your-domain.com
Automatic Renewal: Certbot automatically sets up certificate renewal. Verify with:
sudo systemctl status certbot.timer

Step 5.3: Configure Environment Variables

Create a production environment file:

nano .env.production

Add the following configuration:

NODE_ENV=production PORT=5000 # Database Configuration DATABASE_URL=postgresql://tally_user:your_secure_password_here@localhost:5432/tally_db PGHOST=localhost PGPORT=5432 PGDATABASE=tally_db PGUSER=tally_user PGPASSWORD=your_secure_password_here # Session Configuration SESSION_SECRET=your_very_long_random_session_secret_here # Application Configuration REPL_ID=tally-production REPLIT_DOMAINS=yourdomain.com,www.yourdomain.com ISSUER_URL=https://yourdomain.com/oidc
Security Critical:
  • Replace all password placeholders with your actual secure passwords
  • Generate a strong session secret (at least 64 random characters)
  • Replace yourdomain.com with your actual domain
  • Keep this file secure and never commit it to version control

Step 5.4: Set File Permissions

# Set appropriate permissions chmod 600 .env.production chown tally:tally .env.production # Make sure the tally user owns all application files sudo chown -R tally:tally /home/tally/tally-app

Step 5.5: Initialize Database Schema

# Load environment variables and run database setup export $(cat .env.production | xargs) # Push database schema npm run db:push # If you have seed data, run: # npm run db:seed

Step 5.6: Build the Application

# Build the production version npm run build

This will create optimized production files in the dist directory.

Step 5.7: Test the Application

# Test run the application npm start

You should see output indicating the server is running on port 5000. Press Ctrl+C to stop the test run.

6. Monitoring and Maintenance

Step 6.1: Check Application Status

# Check PM2 processes sudo -u tally pm2 status # Check application logs sudo -u tally pm2 logs tally # Monitor real-time logs sudo -u tally pm2 logs --lines 100

Step 6.3: Enable the Site

# Enable the site sudo ln -s /etc/nginx/sites-available/tally /etc/nginx/sites-enabled/ # Remove default site (optional) sudo rm /etc/nginx/sites-enabled/default # Test Nginx configuration sudo nginx -t # Reload Nginx sudo systemctl reload nginx

Step 6.4: Obtain SSL Certificate

# Get SSL certificate from Let's Encrypt sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

Follow the prompts to complete the SSL certificate setup. Certbot will automatically update your Nginx configuration.

Step 6.5: Test SSL Auto-Renewal

# Test certificate renewal sudo certbot renew --dry-run

This should complete without errors.

7. Process Management with PM2

Step 7.1: Install PM2

# Install PM2 globally sudo npm install -g pm2

Step 7.2: Create PM2 Configuration

Create a PM2 ecosystem file:

nano ecosystem.config.js

Add the following configuration:

module.exports = { apps: [{ name: 'tally', script: 'npm', args: 'start', cwd: '/home/tally/tally-app', env: { NODE_ENV: 'production', }, env_file: '/home/tally/tally-app/.env.production', instances: 1, autorestart: true, watch: false, max_memory_restart: '1G', error_file: '/home/tally/logs/tally-error.log', out_file: '/home/tally/logs/tally-out.log', log_file: '/home/tally/logs/tally-combined.log', time: true }] };

Step 7.3: Create Log Directory

mkdir -p /home/tally/logs

Step 7.4: Start Application with PM2

# Start the application pm2 start ecosystem.config.js # Save PM2 configuration pm2 save # Setup PM2 to start on boot pm2 startup # Follow the instructions provided by the startup command

Step 7.5: PM2 Management Commands

# Check application status pm2 status # View logs pm2 logs tally # Restart application pm2 restart tally # Stop application pm2 stop tally # Monitor application pm2 monit

9. Troubleshooting

Common Issues and Solutions

Application Won't Start

Check:
  • Environment variables are correctly set
  • Database connection is working
  • Port 5000 is not being used by another service
  • All dependencies are installed
Debug commands:
pm2 logs tally npm run build lsof -i :5000

Database Connection Issues

Check:
  • PostgreSQL service is running
  • Database credentials are correct
  • Database exists and user has permissions
Debug commands:
sudo systemctl status postgresql psql -h localhost -U tally_user -d tally_db

SSL Certificate Issues

Check:
  • Domain DNS is pointing to your server
  • Firewall allows HTTP/HTTPS traffic
  • Nginx configuration is correct
Debug commands:
sudo nginx -t sudo certbot certificates nslookup yourdomain.com

File Upload Issues

Check:
  • Nginx client_max_body_size is sufficient
  • Disk space is available
  • Upload directory permissions are correct
Debug commands:
df -h ls -la /home/tally/tally-app/uploads/

Performance Optimization

Database Optimization

# Optimize PostgreSQL for production sudo nano /etc/postgresql/*/main/postgresql.conf # Add these optimizations: # shared_buffers = 256MB # effective_cache_size = 1GB # maintenance_work_mem = 64MB # checkpoint_completion_target = 0.9 # wal_buffers = 16MB # default_statistics_target = 100 # Restart PostgreSQL sudo systemctl restart postgresql

Nginx Optimization

# Add to nginx.conf in http block sudo nano /etc/nginx/nginx.conf # Add these optimizations: # gzip on; # gzip_vary on; # gzip_min_length 1024; # gzip_types text/plain text/css application/json application/javascript text/xml application/xml; # Restart Nginx sudo systemctl restart nginx
